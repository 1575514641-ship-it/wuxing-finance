# 五行理财 App 使用说明

当前版本：v7.22（上班前收官版）。核心定位是「工资到账决策器 + 执行纪律面板」：工资到账后先做本月分配计划，实际买入后再去「随手记」记录执行，计划不会覆盖实际投入。

v7.22 后进入维护期：只修 bug、回填真实工资/税务/保险/资金路径、更新使用说明；不再新增投资逻辑，不再重开资产配置。

PWA 网页 App，已部署公网且配置云同步（Cloudflare Workers + D1），电脑和手机随时都能用。

## 打开方式

**公网入口（推荐）**：[https://www0706.netlify.app/](https://www0706.netlify.app/)

不依赖本机服务，电脑关机或换网络都能用。建议加到主屏幕：

- iPhone Safari：分享 → "添加到主屏幕"
- 安卓 Chrome / 夸克：菜单 → "添加到主屏幕"

**本机调试入口**：

1. 右键运行 `启动App.ps1`
2. 浏览器自动打开 `http://127.0.0.1:8765`
3. 不用了运行 `停止App.ps1`

如果 Windows 阻止运行脚本，在 PowerShell 里执行：
`Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`

## 数据保存

数据同时保存在：

1. 浏览器 localStorage（本地缓存，秒开）
2. Cloudflare D1 云端数据库（跨设备同步，经 Netlify `/api/sync` 反向代理访问，国内手机无需翻墙）

顶栏右上角显示同步状态：`云同步`（已连接）、`同步中`（连接中）、`本地`（未配置云端）、`同步失败`（云端连接异常）。

同步码等于账本钥匙，不要截图发给别人。同步不是备份，每月月底仍建议点一次 `导出` 保存 JSON。

从 v7.12 起：切回页面（从别的标签/App 切回来）会自动拉取云端最新数据；关闭/切走页面会把还没发出的改动补发；断网恢复后自动重试。多设备之间不用手动刷新就能看到对方的最新记录。

## 云同步（已配置）

云同步通过 Cloudflare Workers + D1 实现，配置已内置在 App 中，无需任何手动设置。后端不会因不活跃而暂停。

### 手机端使用

1. 手机浏览器打开公网入口 [https://www0706.netlify.app/](https://www0706.netlify.app/)
2. 电脑端打开同一地址，点右上角 `同步`，复制本机同步码
3. 同步码发到手机（微信/QQ 自己），手机端粘贴到 `导入并同步`

同步码相当于这份账本的钥匙。不要发给别人，不要放到公开文档里。

如果只想在局域网里调试，电脑跑 `启动App.ps1` 后，手机连同一 WiFi 访问 `http://<电脑IP>:8765`，操作流程相同。

注意：本地 `127.0.0.1:8765` 预览不支持 Netlify `/api/sync` 云同步；需要同步时请到公网入口操作。同步失败时本机数据仍在 localStorage，不要反复刷新，先导出 JSON 备份。

### 如需更换云同步后端

后端是 `理财App/cf-worker/` 下的 Cloudflare Worker，数据存在 D1。如需改后端地址，改 `index.html` 里 `supabase.setConfig("/api/sync")` 指向的路径，以及 Netlify `_redirects` 的反向代理目标即可。Worker 代码改动后在 `cf-worker/` 目录跑 `wrangler deploy`。详细运维见 `AGENTS.md`。

### 退回本地模式

控制台执行 `supabase.clearConfig()` 然后刷新（接口名沿用历史命名，实际操作的是 CF 同步配置）。数据仍在 localStorage 中。

### 换手机/换电脑

- 旧设备还能打开：复制旧设备同步码，在新设备导入。
- 旧设备打不开但有 JSON 备份：新设备导入 JSON，再生成新的同步码。
- 点了 `生成新的同步码`：会创建一套新的云端身份，旧设备不会自动跟随；只有在所有设备都导入新同步码后，才会继续同步同一份数据。

## 日常怎么用

### 打开 App 先看「今日动作」

首页资产总览下方会显示「今日动作」，只保留 3-5 条最需要处理的事，例如：

- 本月计划未保存：去分配。
- 计划已保存，买入后记得去随手记。
- 每月 25 号后如果还没填月末资产，会提示月底更新市值。
- 应急金未达标时，本月优先补现金层。
- 投机层达到 10% 时，提醒暂停新增。

今日动作不保存打卡状态，只按当前账本数据实时生成。

首页的「命盘纪律」只保留行为摘要：少操作，先存后花，投机层封顶，月度复盘，不用情绪改配置。App 不保存完整出生时间、地点和农历生日。

### 工资到账后

打开 `分配`，输入月收入、固定支出、机动预留和目标储蓄率。App 会按目标占比和当前持仓偏离情况生成本月计划，点 `保存为本月计划` 后会写入月度计划金额，不会覆盖实际投入。

分配页会同时显示「工资到账后操作清单」：

- 固定支出账户：本月固定支出。
- 机动预留：本月不确定开销。
- 本月计划投资：本月实际建议分配出去的金额。
- 暂存/剩余现金：未分配出去的现金；如果暂存去向不可用，会在这里提示。

国内阶段部分 QDII/海外资产可能显示为「暂存」，资金会先转入指定暂存资产。资产页的货币基金等接收方会显示「含暂存后」有效目标，避免误以为它天然超配。

如果已经出海并开通海外渠道，在「规则」页点 `我已出海·解锁暂存`，可一次性把所有暂存资产改为可买状态。当前市值和累计投入不会被改动。

如果投机层当前市值已经达到或超过总资产 10%，分配页会自动暂停给投机层新增资金。这个规则在标准模式和修正模式下都生效。

分配页还有「机会补仓检查」。它只在市场明显回撤时使用：先在资产页更新现金层市值，再手动勾选已确认的回撤条件。App 会按“现金层超出应急金目标的弹药”计算建议金额；投机层达到 10% 时相关建议自动为 0；buffered 资产会提示“暂存到货币基金”。矿股不列入机会补仓检查器。这个检查器只显示建议，不保存、不执行、不写入随手记。

v7.19 已固化《半解锁与出海执行规则 v1》，冻结至 2026-09-12。冻结期只执行、回填真实入职信息和修 bug，不再反复重开配置比例或规则框架。国内阶段标普500最多先建到目标 12%，全球医疗最多 3%，纳指仍按投机层管理，USD 货币工具继续暂存。QDII 场内溢价小于 1% 正常买，1%-2% 小额买，2%-3% 半额买，大于 3% 暂停，大于 5% 绝不买。

### FIRE 测算

打开 `FIRE`，填写目标年支出、当前年龄、目标年龄、通胀率、最低线实际收益率、半 FIRE 后预计年补充收入、国内阶段月存、国内阶段月数、海外阶段月存和首年安家支出。页面会展示：

- 4% 最低线、3.5% 稳健线、3% 安心线的「今天购买力」本金。
- 到目标年龄时对应的名义金额。
- 当前总资产相对 4% 线的进度。

注意：4% 是提取率，不是简单收益率。页面把「今天购买力」和「目标年龄名义金额」分开显示，避免重复扣通胀。
达成日预测按国内/海外两段月存滚动，默认国内阶段月存 ¥4,000、12 个月，海外阶段月存 ¥15,000，首年安家支出 ¥12,000。入职后用真实工资结构回填。

FIRE 页现在带「参数可信度」提示：真实税后工资、房租、汇率、保险和派驻地确认后，再把它作为正式路线图。

### 平时

打开 `随手记`，只记三类：收入、投资、大额消费。

这里记录真实发生，不记录计划。计划只在分配页保存。

### 每月月底

- `资产`：更新每个产品的当前市值和累计投入
- `月度`：填写月收入、支出、实际投入、月末总资产。若当月已保存分配计划，月度卡片会同时显示计划投资、实际投入；含暂存时会显示「含暂存」金额。
- `月度清单`：检查本月计划、实际投入、月末总资产和应急金目标状态。它是静态核对，不是任务管理器。

### 入职后回填

规则页新增「入职后回填」折叠块，可复制清单。拿到真实信息后，只回填数字和备注，不重开配置框架：

- 国内/海外税后工资和发薪币种。
- 境内/境外发薪比例、奖金结算周期。
- 社保、公积金、境外所得税务安排。
- 公司商业保险、国内百万医疗、境外意外险覆盖范围。
- 海外工资入账银行、是否可直接入金 IB、是否占用 5 万美元购汇额度。
- FIRE 国内阶段月存、海外阶段月存、首年安家支出、应急金目标。

### 备份

每月点一次右上角 `导出` 保存 JSON，双重保险。

## 技术说明

- 前端：纯 HTML/CSS/JS，无框架
- PWA：可安装到桌面，离线可用
- 存储：localStorage（默认）+ Cloudflare D1（云同步，经 Netlify 反代访问）
- 不收集任何数据，所有数据由你控制
